Delegation

Entities act mainly by delegating the work to the grid, state and queue components of the level object. That allows convenience methods to be defined in the Cell class:

class Cell<E> {
    private Level<E> level;

    void wake(E e) { level.grid.wake(...); ... }
}

In fact, by making the level field private, classes which extend Cell are forced to use the convenience methods, which is a good thing because it makes them immune from the details of the level object and its components.

But now we run into a problem. How is the level field in the Cell class to be initialized? The normal way would be through a constructor:

class Cell<E> {
    Cell(Level<E> l) { ... }
}

But that would mean each extending class would have to have a similar constructor:

class Entity extends Cell<Entity> {
    Entity(Level<Entity> l) { super(g...); }
}

That would be ugly because (a) we would like to keep custom entity classes ultra-simple by handling as many details as possible in framework classes and (b) custom entity classes would not be completely independent of the details of how the Cell class works.

A solution is to set the field via a method rather than a constructor:

class Cell<E> {
    void init(Level<E> l) { ... }
}

Now extending classes can be kept simple.
